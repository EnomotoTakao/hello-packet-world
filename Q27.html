Question

You plan to deploy a cloud service to an Azure subsctiption.

You notice that the cloud service has a public IP address.

You need to ensure that the IP address assigned to the cloud service persists for one year.

What should you do first?

---------
Choose the correct answer
---------

[ ] Run the Set-AzureStaticVNetIP cmdlet and then modify the service definition file.
[ ] Run the New-AzureReservedIP cmdlet and then modify the service definition file
[ ] Run the Set-AzureStaticVNetIP cmdlet and then modify the service configuration file.
[ ] Run the New-AzureReservedIP cmdlet and then modify the service configuration file.

Japanese Question

Answer

[ ] Run the Set-AzureStaticVNetIP cmdlet and then modify the service definition file.
[ ] Run the New-AzureReservedIP cmdlet and then modify the service definition file
[ ] Run the Set-AzureStaticVNetIP cmdlet and then modify the service configuration file.
[*] Run the New-AzureReservedIP cmdlet and then modify the service configuration file.

Japanese Answer

Explanation

You should run the New-AzureReservedIP cmdlet. This cmdlet is used to reserve a public IP address. You should then modify the service configuration file to associate the public reserved IP address with the cloud service.

You should not use the service definition file. This file does not support network configuration changes The service definition file defines the service model for a cloud servicec, including definition of its roles and service endpoints, but not the virtual network settings.

You should not run the Set-AzureStaticVNetIP cmdlet. This cmldet is used to configure static IP addresses, not virtual IP (VIP) addresses.
